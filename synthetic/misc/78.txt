How can I use scientific notation in the `\addplot` part? I have some really large numbers.
§a§
### Short answer
You can write the coordinates in *scientific notation* directly, e.g.

```tex
\addplot[only marks,mark=*,mark size=3pt,black] coordinates {
    (1e6, 2.3e-2)   % 1 000 000  on the x‑axis, 0.023 on the y‑axis
    (5e7, 1.1e-1)   % 50 000 000  on the x‑axis, 0.11  on the y‑axis
    (2.5e8, 3e-3)   % 250 000 000 on the x‑axis, 0.003 on the y‑axis
};
```

`pgfplots` (and the underlying `pgfmath`) understands the forms

* `1e6`  (or `1E6`) → 1 × 10⁶
* `2.3e-2` → 2.3 × 10⁻²

No extra package or option is required.

---

## Why it works

`pgfplots` parses the numbers in the *coordinates* with the same parser that `pgfmath` uses.
`pgfmath` accepts the usual C‑style scientific notation (`<mantissa>e<exponent>`).
Therefore the coordinates can be written exactly as you would write them in a normal
calculator.

---

## Full minimal example

Below is a self‑contained document that

* uses a logarithmic *x* axis (`xmode=log`);
* plots three points whose *x* values are in the millions;
* shows how to format the tick labels in scientific notation (optional).

```tex
\documentclass[tikz,border=5pt]{standalone}
\usepackage{pgfplots}
\pgfplotsset{compat=1.18} % use a recent version

\begin{document}
\begin{tikzpicture}
\begin{axis}[
    axis x line=middle,
    axis y line=left,
    xlabel={Number of tokens used in training},
    ylabel={BiasBench score},
    enlargelimits=0.1,
    legend style={
        at={(0.5,-0.18)},anchor=north,
        legend columns=2, column sep=1em,
        draw=none, font=\small
    },
    xmode=log,                     % log scale on the x‑axis
    log ticks with fixed point,    % keep the tick numbers readable
    width=\textwidth,
    height=0.25\textheight,
    y dir=reverse,
    %--- optional: show x‑ticks in scientific notation
    tick label style={/pgf/number format/.cd,
        sci, % use scientific notation
        precision=2, % two significant digits
        fixed zerofill,
    },
]
    \addplot[only marks,mark=*,mark size=3pt,black] coordinates {
        (1e6, 2.3e-2)
        (5e7, 1.1e-1)
        (2.5e8, 3e-3)
    };
    \addlegendentry{My data}
\end{axis}
\end{tikzpicture}
\end{document}
```

### What each part does

| Option | Meaning |
|--------|---------|
| `xmode=log` | Makes the *x* axis logarithmic (useful for huge ranges). |
| `log ticks with fixed point` | Keeps the tick labels as ordinary numbers (e.g. `10⁶` instead of `1e6`). |
| `tick label style={/pgf/number format/.cd, sci, precision=2}` | (optional) forces the tick labels themselves to be printed in scientific notation, with two significant digits. |
| `coordinates { (1e6,2.3e-2) … }` | The actual data points, written in scientific notation. |

## Common pitfalls & how to avoid them

| Symptom | Cause | Fix |
|---------|-------|-----|
| **“Undefined control sequence”** when using `e` notation | You are using a *very old* `pgfplots` version (< 1.3). | Update `pgfplots` (the current CTAN version is 1.18). |
| **Ticks look like “1e+06”** on a linear axis | The axis is linear, so the default tick formatter prints the raw number. | Either switch to `xmode=log` (recommended for huge ranges) **or** add `tick label style={/pgf/number format/.cd, sci}` to force scientific notation on a linear axis. |
| **Numbers are rounded to 0** | The exponent is too large for the default `precision`. | Increase `precision` (e.g. `precision=4`) or use `fixed`/`fixed zerofill` to control formatting. |
| **Coordinates are ignored** | You accidentally placed a semicolon after the `coordinates` block or used a stray comma. | Ensure the block ends with `};` (no extra comma after the last point). |

## Alternative ways to feed large numbers

If you prefer not to write the numbers directly in the source, you can also

1. **Read them from an external file** (`\addplot table {data.txt};`).
   The file may contain scientific notation as well:

   ```
   x y
   1e6 2.3e-2
   5e7 1.1e-1
   2.5e8 3e-3
   ```

2. **Compute them on the fly** with `\pgfmathsetmacro`:

   ```tex
   \pgfmathsetmacro{\x}{1e6}
   \pgfmathsetmacro{\y}{2.3e-2}
   \addplot coordinates { (\x,\y) };
   ```

Both approaches use the same parser, so the scientific notation works identically.

With this i get numbers like $10^{12.00}$. How can change that to $x.y * 10^z$?
§a§
`pgfplots` lets you decide **how** the tick label is formatted.
What you are seeing (`10^{12.00}`) is the *default* scientific-notation style that `pgfplots` uses when you only say `sci`.
To switch to the “× 10^z” form (engineering / SI style) you have to

1. tell `pgfplots` to use the SI style, and
2. (optionally) reduce the number of digits after the decimal point.

Add the following to the axis options:

```tex
x tick label style={
    /pgf/number format/.cd,
    sci,                 % scientific notation
    sci generic={        % <-- this gives “x.y × 10^z”
        mantissa sep=\times,   % separator between mantissa and exponent
        exponent={10^{#1}}     % exponent base
    },
    precision=1,         % one digit after the dot (change as you like)
    zerofill=false,      % suppress trailing zeros
}
```

Complete minimal example:

```tex
\documentclass[tikz,border=5pt]{standalone}
\usepackage{pgfplots}
\pgfplotsset{compat=1.18}

\begin{document}
\begin{tikzpicture}
\begin{axis}[
    xmode=log,
    log ticks with fixed point=false, % we *want* sci labels
    xlabel={Number of tokens},
    ylabel={BiasBench score},
    enlargelimits=0.1,
    width=12cm, height=6cm,
    x tick label style={
        /pgf/number format/.cd,
        sci,
        sci generic={
            mantissa sep=\times,
            exponent={10^{#1}}
        },
        precision=1,
        zerofill=false,
    },
]

\addplot[only marks,mark=*,black] coordinates {
    (1e6, 0.02)
    (5e7, 0.11)
    (2.5e8, 0.003)
};

\end{axis}
\end{tikzpicture}
\end{document}
```

The *x*-axis labels now look like
`1.0 × 10^6`, `5.0 × 10^7`, `2.5 × 10^8` …
instead of the awkward `10^{6.00}`.

If you prefer a dot instead of the multiplication sign, replace `\times` with `\cdot`.
