The issue is that your Minecraft server failed to start because the port it's trying to use (25565) is already in use by another application. This is indicated by the error:

```
[21:20:54] [Server thread/WARN] [minecraft/DedicatedServer]: **** FAILED TO BIND TO PORT!
[21:20:54] [Server thread/WARN] [minecraft/DedicatedServer]: The exception was: io.netty.channel.unix.Errors$NativeIoException: bind(..) failed: Address already in use
[21:20:54] [Server thread/WARN] [minecraft/DedicatedServer]: Perhaps a server is already running on that port?
```

### Solutions:

1. **Check for another running Minecraft server**:
   - You might already have another Minecraft server instance running on the same port.
   - Use commands like `netstat -tulnp | grep 25565` (Linux) or `netstat -ano | findstr 25565` (Windows) to check which process is using the port.

2. **Change the server port**:
   - Edit your `server.properties` file and change `server-port=25565` to another unused port (e.g., `25566`).

3. **Kill the existing process** (if safe to do so):
   - If another Minecraft server is running, stop it properly.
   - If it's an orphaned process, kill it manually (e.g., `kill -9 <PID>` on Linux or via Task Manager on Windows).

4. **Wait a few minutes**:
   - Sometimes, the port remains locked briefly after a server crash. Wait a minute and try again.

### Additional Notes:
- The `NullPointerException` during shutdown is a secondary issue caused by the server failing to start properly.
- The warning `No specified tag 'minecraft:carpets' doesn't exist!` is unrelated and can be ignored for now.

Try one of the solutions above, and your server should start normally. Let me know if you need further help!
